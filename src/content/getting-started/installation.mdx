# Installation

This guide covers the installation of Xether AI platform components, SDKs, and development tools. Choose the installation method that best fits your use case.

## System Requirements

### Minimum Requirements

**Development Environment**
- CPU: 2 cores
- Memory: 4GB RAM
- Storage: 10GB available space
- Network: Internet connection for SDK downloads

**Production Environment**
- CPU: 4 cores (8+ recommended)
- Memory: 8GB RAM (16GB+ recommended)
- Storage: 100GB SSD (500GB+ recommended)
- Network: 1Gbps connection

### Supported Platforms

**Operating Systems**
- Linux: Ubuntu 20.04+, CentOS 8+, RHEL 8+
- macOS: 11.0+ (Big Sur)
- Windows: 10/11 (with WSL2 recommended)

**Container Platforms**
- Docker 20.10+
- Kubernetes 1.24+
- Podman 3.0+

**Cloud Providers**
- AWS: ECS, EKS, EC2
- Google Cloud: GKE, Cloud Run
- Azure: AKS, Container Instances

## SDK Installation

### Python SDK

#### Prerequisites
- Python 3.8 or higher
- pip package manager
- Virtual environment (recommended)

#### Installation Methods

**Standard Installation**
```bash
pip install xether-ai
```

**With Development Dependencies**
```bash
pip install xether-ai[dev]
```

**With All Optional Dependencies**
```bash
pip install xether-ai[all]
```

**From Source**
```bash
git clone https://github.com/xether-ai/python-sdk.git
cd python-sdk
pip install -e .
```

**Verification**
```python
import xether_ai
print(f"Xether AI SDK version: {xether_ai.__version__}")

# Test connection
client = xether_ai.Client(api_key="your-api-key")
datasets = client.datasets.list()
print(f"Connected successfully! Found {len(datasets)} datasets")
```

## Platform Installation

### Docker Installation

#### Prerequisites
- Docker 20.10+
- Docker Compose 2.0+

#### Quick Start

**Using Docker Compose**
```bash
# Clone the repository
git clone https://github.com/xether-ai/platform.git
cd platform

# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f
```

**Individual Service Installation**
```bash
# API Gateway
docker run -d \
  --name xether-api-gateway \
  -p 8080:8080 \
  -e API_KEY=your-api-key \
  xether-ai/api-gateway:latest

# Pipeline Service
docker run -d \
  --name xether-pipeline-service \
  -p 8081:8080 \
  -e DATABASE_URL=postgresql://user:pass@localhost:5432/xether \
  xether-ai/pipeline-service:latest

# Execution Service
docker run -d \
  --name xether-execution-service \
  -p 8082:8080 \
  -e QUEUE_URL=redis://localhost:6379 \
  xether-ai/execution-service:latest
```

#### Docker Compose Configuration

```yaml
# docker-compose.yml
version: '3.8'

services:
  api-gateway:
    image: xether-ai/api-gateway:latest
    ports:
      - "8080:8080"
    environment:
      - API_KEY=${XETHER_AI_API_KEY}
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/xether
      - REDIS_URL=redis://redis:6379
    depends_on:
      - postgres
      - redis
    restart: unless-stopped

  pipeline-service:
    image: xether-ai/pipeline-service:latest
    ports:
      - "8081:8080"
    environment:
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/xether
      - KAFKA_BROKERS=kafka:9092
    depends_on:
      - postgres
      - kafka
    restart: unless-stopped

  execution-service:
    image: xether-ai/execution-service:latest
    ports:
      - "8082:8080"
    environment:
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/xether
      - REDIS_URL=redis://redis:6379
      - KAFKA_BROKERS=kafka:9092
    depends_on:
      - postgres
      - redis
      - kafka
    restart: unless-stopped

  ml-service:
    image: xether-ai/ml-service:latest
    ports:
      - "8083:8080"
    environment:
      - DATABASE_URL=postgresql://postgres:password@postgres:5432/xether
      - MODEL_STORAGE_PATH=/models
    volumes:
      - ./models:/models
    depends_on:
      - postgres
    restart: unless-stopped

  postgres:
    image: postgres:14
    environment:
      - POSTGRES_DB=xether
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped

  kafka:
    image: confluentinc/cp-kafka:latest
    environment:
      KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
      KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:9092
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
    ports:
      - "9092:9092"
    depends_on:
      - zookeeper
    restart: unless-stopped

  zookeeper:
    image: confluentinc/cp-zookeeper:latest
    environment:
      ZOOKEEPER_CLIENT_PORT: 2181
      ZOOKEEPER_TICK_TIME: 2000
    ports:
      - "2181:2181"
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

### Kubernetes Installation

#### Prerequisites
- Kubernetes 1.24+
- kubectl configured
- Helm 3.0+ (recommended)

#### Installation with Helm

**Add Helm Repository**
```bash
helm repo add xether-ai https://charts.xether.ai
helm repo update
```

**Install Platform**
```bash
# Create namespace
kubectl create namespace xether-ai

# Install with default values
helm install xether-platform xether-ai/xether-platform \
  --namespace xether-ai

# Install with custom values
helm install xether-platform xether-ai/xether-platform \
  --namespace xether-ai \
  --values custom-values.yaml
```

**Custom Values Example**
```yaml
# custom-values.yaml
apiGateway:
  replicaCount: 3
  resources:
    requests:
      memory: "256Mi"
      cpu: "250m"
    limits:
      memory: "512Mi"
      cpu: "500m"

pipelineService:
  replicaCount: 2
  resources:
    requests:
      memory: "512Mi"
      cpu: "500m"
    limits:
      memory: "1Gi"
      cpu: "1000m"

executionService:
  replicaCount: 5
  resources:
    requests:
      memory: "1Gi"
      cpu: "1000m"
    limits:
      memory: "2Gi"
      cpu: "2000m"

postgresql:
  enabled: true
  auth:
    postgresPassword: "secure-password"
    database: "xether"

redis:
  enabled: true
  auth:
    password: "secure-password"

kafka:
  enabled: true
  replicaCount: 3
```

**Namespace and ConfigMaps**

**Secrets**
```yaml
# secrets.yaml

kind: Secret
metadata:
  name: xether-secrets
  namespace: xether-ai
type: Opaque
data:
  API_KEY: <base64-encoded-api-key>
  DATABASE_PASSWORD: <base64-encoded-password>
  REDIS_PASSWORD: <base64-encoded-password>
```

**Deployments**
```yaml
# api-gateway-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api-gateway
  namespace: xether-ai
spec:
  replicas: 3
  selector:
    matchLabels:
      app: api-gateway
  template:
    metadata:
      labels:
        app: api-gateway
    spec:
      containers:
      - name: api-gateway
        image: xether-ai/api-gateway:latest
        ports:
        - containerPort: 8080
        env:
        - name: API_KEY
          valueFrom:
            secretKeyRef:
              name: xether-secrets
              key: API_KEY
        envFrom:
        - configMapRef:
            name: xether-config
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5

---
# api-gateway-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: api-gateway
  namespace: xether-ai
spec:
  selector:
    app: api-gateway
  ports:
  - port: 80
    targetPort: 8080
  type: LoadBalancer
```

## Development Setup

### Local Development Environment

#### Prerequisites
- Docker Desktop
- Node.js 16+
- Python 3.8+
- Go 1.19+

#### Setup Script

```bash
#!/bin/bash
# setup-dev.sh

echo "Setting up Xether AI development environment..."

# Check prerequisites
command -v docker >/dev/null 2>&1 || { echo "Docker is required but not installed."; exit 1; }
command -v node >/dev/null 2>&1 || { echo "Node.js is required but not installed."; exit 1; }
command -v python3 >/dev/null 2>&1 || { echo "Python 3 is required but not installed."; exit 1; }

# Clone repositories
echo "Cloning repositories..."
git clone https://github.com/xether-ai/platform.git
git clone https://github.com/xether-ai/python-sdk.git
git clone https://github.com/xether-ai/javascript-sdk.git
git clone https://github.com/xether-ai/go-sdk.git

# Setup Python SDK
echo "Setting up Python SDK..."
cd python-sdk
python3 -m venv venv
source venv/bin/activate
pip install -e .[dev]
cd ..

# Setup JavaScript SDK
echo "Setting up JavaScript SDK..."
cd javascript-sdk
npm install
npm run build
cd ..

# Setup Go SDK
echo "Setting up Go SDK..."
cd go-sdk
go mod download
go install ./...
cd ..

# Start platform services
echo "Starting platform services..."
cd platform
docker-compose up -d

# Wait for services to be ready
echo "Waiting for services to be ready..."
sleep 30

# Run health check
echo "Running health check..."
curl -f http://localhost:8080/health || echo "API Gateway not ready yet"

echo "Development environment setup complete!"
echo "API Gateway: http://localhost:8080"
echo "Pipeline Service: http://localhost:8081"
echo "Execution Service: http://localhost:8082"
echo "ML Service: http://localhost:8083"
```

## Troubleshooting

### Common Issues

#### Installation Failures

**Permission Denied (Linux/macOS)**
```bash
# Fix permissions
sudo chown -R $USER:$USER ~/.xether-ai

# Or use virtual environment
python3 -m venv ~/.xether-ai/venv
source ~/.xether-ai/venv/bin/activate
pip install xether-ai
```

**Network Connection Issues**
```bash
# Check internet connection
curl -I https://api.xether.ai

# Use proxy if needed
export HTTPS_PROXY=http://proxy.company.com:8080
export HTTP_PROXY=http://proxy.company.com:8080
pip install xether-ai
```

#### Docker Issues

**Port Conflicts**
```bash
# Check port usage
netstat -tulpn | grep :8080

# Use different ports
docker-compose up -d --scale api-gateway=1 -p 8080:8080
```

**Resource Limits**
```bash
# Check Docker resources
docker system df
docker system prune

# Increase Docker memory allocation in Docker Desktop settings
```

#### Kubernetes Issues

**Pod Not Starting**
```bash
# Check pod status
kubectl get pods -n xether-ai

# Check pod logs
kubectl logs -f deployment/api-gateway -n xether-ai

# Describe pod for detailed information
kubectl describe pod <pod-name> -n xether-ai
```

**Service Not Accessible**
```bash
# Check service status
kubectl get services -n xether-ai

# Check endpoints
kubectl get endpoints -n xether-ai

# Port-forward for testing
kubectl port-forward service/api-gateway 8080:80 -n xether-ai
```
